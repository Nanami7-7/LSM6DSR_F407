# BSP_LSM6DSR 调参与使用指南

## 1. 概述

本指南适用于机器狗 IMU 姿态估计场景，基于增强型互补滤波器：

- 方差滑动窗口静止检测（抗振动干扰）
- 自适应 α（运动 0.99 / 静止 0.30）
- 快速偏置跟踪（站立时速率 0.05）

### 架构

```
┌─ main.c ──────────────────────────────────┐
│  run_all_tests() / phase17_live_display()  │
├───────────────────────────────────────────┤
│  test_lsm6dsr.c   测试层 (P1-P19)          │
│  bsp_lsm6dsr.c    业务层 (滤波器核心)       │
│  lsm6dsr.c        驱动层 (寄存器读写)       │
└───────────────────────────────────────────┘
```

## 2. 参数详解

### 2.1 静止检测：方差滑动窗口

```c
BSP_ACC_VAR_WINDOW      10       // 窗口大小（帧），越大越平滑
BSP_ACC_VAR_THRESHOLD   800.0    // 方差阈值（mg²），低于此值判静止
```

**原理**：计算过去 N 帧 ACC 的 3 轴方差总和。静止时方差小（只有传感器噪声 ~1-5 mg²），行走时方差大（冲击 ~10000+ mg²）。

**对比帧间差分**：
- 帧间差分比较当前帧和上一帧，机器狗马达振动使每帧都不同 → 永远判为运动
- 方差窗口看统计分布，振动噪声被平均掉

### 2.2 自适应 α

```c
BSP_ALPHA_MOVING        0.99f    // 运动时 α——1% ACC 权重
BSP_ALPHA_STATIONARY    0.30f    // 静止时 α——70% ACC 权重
BSP_ALPHA_SMOOTH_STEP   0.15f    // α 切换时每帧最大变化
```

**α 的物理意义**：

```
α = 0.99 → τ = α/(1-α) × 0.009s = 0.89s 时间常数
  → 运动时几乎纯 GYRO，线性加速几乎不污染角度

α = 0.30 → τ = 0.30/0.70 × 0.009s = 0.004s 时间常数
  → 静止时 ACC 快速校正，<0.1s 收敛到真值
```

**平滑过渡**：从 0.99→0.30 需约 5 帧(45ms)，不会角度跳跃。

### 2.3 偏置跟踪

```c
BSP_BIAS_STATIONARY_RATE  0.05f   // 静止时偏置更新速率
```

对比原来的 0.001：快了 50 倍。站立 3s 后可校正大部分温漂偏置。

数学上：
- 每帧：`bgx += 0.05 × (fgx - bgx)` → fgx 是偏置补偿前的值
- 等效低通滤波器时间常数 τ = dt / rate = 9ms / 0.05 = 180ms
- 3s 内收敛到稳定值

### 2.4 陀螺运动阈值

```c
BSP_GYRO_MOTION_THRESHOLD   5.0f    // dps — 超过此值判定为运动
```

**作用**：在 ACC 方差 + mag² 双检基础上增加第三重防护——陀螺仪模值检查。

**为什么需要**：纯 Z 轴旋转时 ACC 不变化，方差和 mag² 都检测不到。如果没有此检查，偏置追踪会把实际旋转当作偏置吃掉，导致 yaw 不跟随旋转（见第 6 节常见问题）。

**工作原理**：
```
gyro_mag = sqrt(fgx² + fgy² + fgz²)
if gyro_mag > BSP_GYRO_MOTION_THRESHOLD → 强制设为"运动"状态
  → alpha 保持 0.99（近纯陀螺）
  → 偏置不更新（防止吃掉旋转）
  → 输出不衰减
```

**阈值选 5.0 dps 的考量**：

| 场景 | 典型陀螺值 | 与阈值关系 |
|------|-----------|-----------|
| 静止（噪声） | < 1 dps | 低于阈值 → 静止 ✅ |
| 温漂（10°C变化） | < 1 dps | 低于阈值 → 可跟踪偏置 ✅ |
| 电机振动（站立） | 1~3 dps | 低于阈值 → 仍判静止 ✅ |
| 极慢旋转（4°/s） | 4 dps | 低于阈值 → 轻微偏置收敛（可接受） ⚠️ |
| 慢旋转（10°/s） | 10 dps | 超过阈值 → 运动 ✅ |
| 快旋转（90°/s） | 90 dps | 超过阈值 → 运动 ✅ |

### 2.5 保留的校准宏

```c
BSP_CALIB_SAMPLES       100      // 启动校准采样数
BSP_CALIB_SETTLE_MS     50       // 配置后稳定延时
BSP_CALIB_ACC_MAG_REF   1000000.0f  // 1G 的 mg² 参考值
BSP_CALIB_ACC_MAG_TOL   65000.0f    // mag 容差
```

启动校准仍然执行 100 次采样，在静止状态下建立初始偏置。运行时跟踪在此基础上微调。

## 3. 首次调参流程

### 3.0 准备工作

1. 串口连接：115200 8N1
2. 上位机：**VOFA+**（推荐）或任何串口终端
3. VFOA+ 配置：
   - Protocol → **FireWater**（浮点逗号分隔）
   - 9 通道顺序：`ax, ay, az, gx, gy, gz, pitch, roll, yaw`
4. 确认 `main.c` 调用 `run_all_tests()`

### 3.1 编译烧录

```
1. 确认 bsp_lsm6dsr.c 在 MDK 工程树中
2. 编译，确认 0 Error 0 Warning
3. 烧录到机器狗
```

### 3.2 运行 P19 Part A（静止漂移测试）

**操作**：
1. 机器狗放在水平地面，保持静止
2. 打开串口终端，观察输出
3. 测试持续 ~13s（1000 帧 × 9ms + 初始化 ~3s）

**预期输出示例**：

```
Calibrating gyro bias (100 samples, keep still)...
Gyro bias: X=0.0234 Y=-0.0156 Z=0.0089 dps (98/100 OK)
BSP init done  (cal=OK, bias=0.0234,-0.0156,0.0089)  alpha=0.30

[P19] Attitude Performance Test
  Part A: 10-second stationary drift
  [   0] p=0.12 r=-0.08 y=0.00  var=2
  [ 200] p=0.14 r=-0.06 y=0.02  var=3
  [ 400] p=0.11 r=-0.07 y=0.04  var=2
  [ 600] p=0.13 r=-0.09 y=0.05  var=3
  [ 800] p=0.12 r=-0.07 y=0.07  var=2

  ==== Results (10s stationary) ====
  Pitch: 0.125±0.015 deg
  Roll:  -0.073±0.012 deg
  Yaw:   0.042±0.018 deg  drift=0.252 deg/min
  Gyro residual: X=0.0002 Y=0.0001 Z=0.0001 dps
  Runtime bias: X=0.0235 Y=-0.0157 Z=0.0091 dps
```

**关键指标**：

| 指标 | 目标 | 本例值 | 判定 |
|------|------|--------|------|
| Yaw 漂移率 | < 2.0°/min | 0.252°/min | ✅ |
| Pitch 标准差 | < 0.8° | 0.015° | ✅ |
| Roll 标准差 | < 0.8° | 0.012° | ✅ |

如果任何一个 FAIL，检查机器人是否在测试期间真的静止，或阈值需要调整。

### 3.3 调整方差阈值

P19 每 200 帧打印 `var=XXX`，这是当前 3 轴方差总和。

| 机器狗状态 | 典型方差范围 | 判定 |
|-----------|------------|------|
| 静止（电机 idle） | 1 ~ 100 | 应判为静止 |
| 静止（电机运转） | 50 ~ 500 | 仍应判静止 |
| 慢走 | 5000 ~ 50000 | 应判为运动 |
| 小跑 | 50000+ | 应判为运动 |

**调整方法**：

```
BSP_ACC_VAR_THRESHOLD = (最大值静止方差 + 最小值行走方差) / 2

例：
  最大站立方差 = 200
  最小行走方差 = 8000
  阈值 = (200 + 8000) / 2 = 4100

改了阈值 → 重新编译 → 重新跑 P19
```

**经验值**：

| 场景 | 推荐阈值 |
|------|---------|
| 纯静止桌面测试 | 200 ~ 500 |
| 机器狗电机 idle | 500 ~ 2000 |
| 机器狗 + 冷却风扇 | 1000 ~ 3000 |
| 不确定时保守值 | 3000 |

保守（偏大阈值）意味着静判定更严格，运动判定更宽松——更安全。

### 3.4 Part B：A→B→A 往返精度

**操作**：
1. 修改 `main.c`：注释 `run_all_tests()`，取消注释 `phase17_live_display()`
2. 编译烧录
3. 打开 VFOA+，配置 9 通道 FireWater
4. 观察波形，手动操作：

```
静止 3s → 绕一个轴转 90° → 静止 3s → 转回 0° → 静止 3s
```

**预期**：
- 旋转中角度平稳跟踪
- 回到 0° 后 pitch/roll 快速收敛到 ±0.3° 以内
- yaw 回零误差 ±1° 以内（短期，长期漂移另计）

**VFOA+ 通道建议**：

```
CH1: pitch（蓝色实线） + acc_pitch（灰色虚线 10% 透明度）
CH2: roll（红色实线） + acc_roll（灰色虚线 10% 透明度）
CH3: yaw（绿色实线）
CH4-6: gx, gy, gz
CH7-9: ax, ay, az 或根据需求选择
```

**通过/失败判断**：

```
回零误差:
  pitch < 0.3°  ✅
  roll  < 0.3°  ✅
  yaw   < 1.0°  ✅
  
  如果 α 切换时有角度跳跃 → 检查 BSP_ALPHA_SMOOTH_STEP 够不够大
  如果回到 0° 后收敛慢 → 减小 BSP_ALPHA_STATIONARY
```

### 3.5 行走测试

**操作**：
1. 保持 VOFA+ 连接
2. 让机器狗慢走 5 米 → 停止 → 观察角度是否回零

**关注**：
- 行走中 pitch/roll 是否大幅偏离（>10° 的可能是线性加速污染）→ 增大 BSP_ALPHA_MOVING
- 停止后是否在 ~0.5s 内收敛 → 减小 BSP_ALPHA_STATIONARY
- 长时间行走后 yaw 漂移量 → 确认偏置跟踪在工作

## 4. VOFA+ 详细使用

### 4.1 FireWater 协议格式

每行 9 个浮点数，逗号分隔，回车换行结尾：

```
-0.863,0.196,-9.413,-0.234,0.156,0.089,84.12,1.85,-2450.61
```

| 通道 | 字段 | 单位 | 精度 |
|------|------|------|------|
| CH1 | ax | m/s² | 3 位小数 |
| CH2 | ay | m/s² | 3 位小数 |
| CH3 | az | m/s² | 3 位小数 |
| CH4 | gx | dps | 3 位小数 |
| CH5 | gy | dps | 3 位小数 |
| CH6 | gz | dps | 3 位小数 |
| CH7 | pitch | ° | 2 位小数 |
| CH8 | roll | ° | 2 位小数 |
| CH9 | yaw | ° | 2 位小数 |

### 4.2 波形设置建议

在 VOFA+ 中：
1. 打开 Data Viewer
2. 选择 Waveform 模式
3. 对每个通道设置颜色和 Y 轴范围：

```
CH1-3 (加速度): 范围 -20 ~ 20, 颜色: 蓝/红/绿
CH4-6 (角速度): 范围 -300 ~ 300, 颜色: 青/紫/黄
CH7-8 (俯仰/横滚): 范围 -90 ~ 90, 颜色: 蓝实线/红实线
CH9 (偏航):    范围 -180 ~ 180, 颜色: 绿实线
```

### 4.3 常见问题

**Q: 波形出现缺口/断续？**
A: 串口缓冲区溢出。降低波特率或增加输出间隔。

**Q: 角度数值突然跳到 180/-180？**
A: 欧拉角 ±180° 跳变，属正常现象。VOFA+ 中设置 yaw 通道用"unwrap"模式。

**Q: 只看到 0,0,0,0,0,0,0,0,0？**
A: I2C 通信失败。检查接线和地址（0x6A）。

## 5. 机器狗使用建议

### 5.1 启动流程

```
1. 上电
2. BSP init 自动执行（~3 秒：复位 + 配置 + 100 帧校准）
3. 听到或看到 "BSP init done" 前保持静止
4. 开始运动
```

### 5.2 长时间运行注意事项

- **偏置跟踪**在站立时自动工作。行走中偏置不更新，所以偶尔静止几秒有助于校正。
- **温度漂移**：芯片温度变化 10°C 可导致偏置漂移 ~0.5 dps → yaw 漂移大幅增加。启动校准只覆盖当时温度。
- **建议策略**：每 5-10 分钟让狗静止 5 秒，让运行时跟踪更新偏置。

### 5.3 参数速查表

| 宏名 | 默认值 | 推荐范围 | 调大方向 | 调小方向 |
|------|--------|---------|---------|---------|
| `BSP_ACC_VAR_THRESHOLD` | 800 | 200~5000 | 狗振动大→调大 | 狗很平滑→调小 |
| `BSP_ALPHA_MOVING` | 0.99 | 0.95~0.999 | 线性加速大→调大 | 响应要快→调小 |
| `BSP_ALPHA_STATIONARY` | 0.30 | 0.10~0.50 | 角度噪声大→调大 | 收敛要快→调小 |
| `BSP_ALPHA_SMOOTH_STEP` | 0.15 | 0.05~0.50 | 过渡不平滑→调小 | 切换要快→调大 |
| `BSP_BIAS_STATIONARY_RATE` | 0.05 | 0.01~0.20 | 偏置漂移快→调大 | 偏置稳定→调小 |
| `BSP_GYRO_MOTION_THRESHOLD` | 5.0 | 3.0~15.0 | 振动导致误触发→调大 | 极慢旋转漏检→调小 |
| `BSP_ACC_VAR_WINDOW` | 10 | 5~20 | 振动慢（如走路）→调大 | 振动快（如电机嗡鸣）→调小 |

## 6. 常见问题

### Q: 慢慢旋转 yaw 时角度几乎不变？

这是偏置追踪在"吃掉"旋转。原因：纯 Z 轴旋转不改变 ACC，因此 ACC 方差和 mag² 都检测不到运动，系统误判为静止，偏置追踪把旋转当偏置收敛。

**解决办法**：`BSP_GYRO_MOTION_THRESHOLD` 已在 v2 中新增，默认 5.0 dps。只要旋转速度 > 5 dps，系统就能正确识别为运动状态。如果仍然有问题（极慢旋转 < 5 dps），可临时调低该阈值进行测试。

### Q: P19 PASS 但实际使用效果差？

阈值是在静止条件下调的，实际运动中振动特性可能不同。在狗实际工作状态下再跑一次 P19，或通过 VOFA+ 观察方差值（后续可增加方差实时输出）。

### Q: 转到 90° 时 pitch/roll 跳变？

这是欧拉角的万向节锁问题（gimbal lock）。当 pitch 接近 ±90° 时，roll 和 yaw 的边界模糊。如果机器狗有爬陡坡或翻滚场景，考虑升级为四元数姿态表示。

### Q: yaw 还在漂？

**6 轴 IMU 没有磁力仪，yaw 始终会漂**。这是物理限制，任何滤波器都无法解决。只需要做到：
- 启动校准移除初始偏置
- 运行时跟踪补偿温漂
- 预期 yaw 漂移率 < 1.0°/min

### Q: 行走中角度突然跳变？

有两种可能：
1. α 从 0.30→0.99 切换时，如果狗刚好在运动中，切换瞬间 α 的错误 ACC 校正注入。增大 `BSP_ALPHA_SMOOTH_STEP` 缓解。
2. 线性加速度进入滤波器。即使 α=0.99，1% 的 ACC 权重在 3G 冲击下也有 ~0.03G 错误注入 → 几度的角度误差。在剧烈运动时可考虑 α=0.999。

### Q: 如何获取更详细的运行时数据？

P19 排印了方差值。如需实时方差，可修改 `phase17_live_display()` 将方差作为第 10 个通道输出，或修改 `bsp_lsm6dsr_data_t` 结构体包含方差字段。

## 7. 快速参考

### 在 main.c 中切换模式

```c
int main(void)
{
    HAL_Init(); SystemClock_Config();
    MX_GPIO_Init(); MX_I2C1_Init(); MX_USART1_UART_Init();

    run_all_tests();                     // P1-P19 全量测试
    // phase17_live_display(&lsm6dsr_io);  // VOFA+ 实时输出

    while (1) { ; }
}
```

### 在你的业务代码中使用

```c
#include "bsp_lsm6dsr.h"

void app_process(void)
{
    bsp_lsm6dsr_init();

    bsp_lsm6dsr_data_t d;
    while (1) {
        bsp_lsm6dsr_update(&d);
        printf("%.2f,%.2f,%.2f\r\n", d.pitch, d.roll, d.yaw);
        HAL_Delay(10);
    }
}
```
